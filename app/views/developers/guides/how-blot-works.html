How Blot works

In essence, Blot is a static file server. It serves your folder to your readers. When a request from one of your readers arrives at your site, Blot follows these steps to work out what to send in response.

  1. Check if there is a template whose route matches the request’s path. Render the template if it matches.
  2. Check if there is a file which matches the request’s path in your site’s folder. Send the file as a response if it matches.
  3. Check your site's list of redirects.
  4. Render your site’s error page if it exists, if not render Blot's error page.

What if a post and a file share the same URL?

Blot shows the post instead of the file. This is because Blot routes requests from your readers to your site’s template first. If you prefer, you can disable your site’s template on the dashboard and just use Blot as a static file server.

Rendering

Blot's templates are written in Mustache. Consider this example template, which produces a list of all of the posts on your site:


  
  {{title}}
  
    {{#all_posts}}
    {{title}}

    {{/all_posts}}

Blot generates JSON containing the data for this template:


  {
    "title": "Your site's title",
    "all_posts": [
    {"title": "First post", "url": "/first-post"},
    {"title": "Second post", "url": "/second-post"},
    {"title": "Third post", "url": "/third-post"}
    ]
  }

Blot then renders the template to produce this HTML:


  
  Your site's title
  
    First post

    Second post

    Third post

Retrieving the JSON used to render a template

If you append the query string ?json=true to the URL of any page on your site, you can see the JSON used to render its template.

Using static files in your templates

Remember that your site’s folder behaves like a static file server. This makes it possible to share assets between your templates.
