Troubleshooting

Tips for debugging templates and understanding what Blot is passing to your views.

Inspect the data

Append ?json=true to any page URL on your blog (e.g. https://yourhandle.blot.im/ or https://yourhandle.blot.im/some-post). Blot will respond with the exact JSON context used to render that page—the same object your Mustache template receives as res.locals—instead of the rendered HTML.

Use it to:

  • See which properties exist on blog, entry, entries, etc.
  • Check the shape of nested data (e.g. thumbnail.medium.url, metadata).
  • Confirm that the data you expect is present before writing or debugging a view.

See template errors on the preview subdomain

When you’re editing a template, use the preview subdomain to see your changes and any render errors. The preview URL is shown in the template editor and looks like:

https://preview-of-template-on-yourhandle.blot.im

For a template you’ve duplicated (your own copy), it will look like:

https://preview-of-my-template-on-yourhandle.blot.im

If Blot fails to render a view—for example because of a syntax error, a missing partial, or an invalid reference—it does not return a generic error page. On the preview subdomain it returns a dedicated error page that lists which template file failed and the error message. Fix the listed view and reload; once the template renders successfully, the error page is replaced by the normal output.

Your live site is never shown these template errors; only the preview subdomain surfaces them.

Getting help

You are always welcome to contact us with your questions. If there's a chance your question might be useful to other template developers, feel free to post it in Questions.
